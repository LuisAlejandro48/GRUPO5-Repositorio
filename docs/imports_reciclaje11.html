Imports del proyecto Reciclaje11

Este documento explica para qué sirve cada import usado en los archivos Java del proyecto. Está separado por carpeta y por archivo para que sea fácil ubicarlo en Android Studio.

Carpeta: app/src/main/java/com/example/reciclaje11

CameraActivity.java

android.content.Intent: abre otra pantalla. Aquí se usa para ir a ResultActivity después de tomar una foto.

android.graphics.Bitmap: representa imágenes en memoria. Se usa para procesar frames y fotos.

android.graphics.BitmapFactory: convierte bytes de imagen en un Bitmap.

android.graphics.ImageFormat: indica el formato de imagen. Se usa con YuvImage para trabajar con NV21.

android.graphics.Matrix: rota el bitmap cuando la cámara entrega la imagen con orientación diferente.

android.graphics.Rect: guarda el rectángulo del objeto detectado y áreas de recorte.

android.graphics.YuvImage: permite convertir un frame YUV de cámara a JPEG.

android.media.Image: obtiene la imagen real desde ImageProxy para enviarla a ML Kit.

android.os.Bundle: recibe datos básicos del ciclo de vida en onCreate.

android.widget.Toast: muestra mensajes cortos como errores de cámara o guardado.

androidx.annotation.NonNull: marca parámetros que no deben ser nulos, como callbacks de CameraX.

androidx.annotation.OptIn: permite usar APIs experimentales de CameraX de forma explícita.

androidx.appcompat.app.AppCompatActivity: base de la Activity con compatibilidad de AppCompat.

androidx.camera.core.CameraSelector: selecciona la cámara, en este caso la trasera.

androidx.camera.core.ExperimentalGetImage: anotación necesaria para usar proxy.getImage().

androidx.camera.core.ImageAnalysis: analiza frames en tiempo real.

androidx.camera.core.ImageCapture: captura fotos desde la cámara.

androidx.camera.core.ImageCaptureException: maneja errores al tomar una foto.

androidx.camera.core.ImageProxy: representa cada frame o foto que entrega CameraX.

androidx.camera.core.Preview: conecta la vista previa de cámara con el PreviewView.

androidx.camera.lifecycle.ProcessCameraProvider: une cámara, preview, captura y análisis al ciclo de vida de la pantalla.

androidx.core.content.ContextCompat: da el executor principal y ayuda con compatibilidad AndroidX.

com.example.reciclaje11.databinding.ActivityCameraBinding: clase generada por ViewBinding para acceder a las vistas de activity_camera.xml.

com.google.mlkit.vision.common.InputImage: formato de imagen que ML Kit necesita para analizar un frame.

com.google.mlkit.vision.objects.DetectedObject: representa un objeto encontrado por ML Kit, con cuadro y etiquetas.

com.google.mlkit.vision.objects.ObjectDetection: crea el detector de objetos de ML Kit.

com.google.mlkit.vision.objects.ObjectDetector: objeto que ejecuta la detección en cámara.

com.google.mlkit.vision.objects.defaults.ObjectDetectorOptions: configura ML Kit en modo tiempo real, múltiples objetos y clasificación.

java.io.ByteArrayOutputStream: guarda temporalmente bytes al convertir YUV a JPEG.

java.io.FileOutputStream: escribe la foto capturada en almacenamiento interno.

java.nio.ByteBuffer: lee bytes desde planos de imagen de CameraX.

java.util.Collections: crea listas simples para el overlay, como una sola caja o una lista vacía.

java.util.List: maneja listas de objetos detectados.

java.util.Locale: formatea textos con porcentaje y traduce etiquetas sin depender del idioma del teléfono.

java.util.concurrent.ExecutorService: ejecuta el análisis de cámara fuera del hilo principal.

java.util.concurrent.Executors: crea el executor usado por CameraX.

com.google.mlkit.vision.common.InputImage: imagen de entrada para ML Kit.

com.google.mlkit.vision.objects.DetectedObject: resultado de detección de ML Kit.

java.util.Locale: formatea porcentajes y compara etiquetas.
